## 跨模块权限设置

比如现在有两个模块：
- MODULE A 中包含了 `security/RBAC_RESOURCE/安全管理/权限设置/权限目标资源/PERM.yml` 文件，但这个目录除了此文件都是空的。
- MODULE B 中包含了 `security/RBAC_RESOURCE/安全管理/权限设置/权限目录资源` 目录，但没有 PERM.yml 文件，内部却包含了如 `菜单权限@GET@_api_acl-menus.yml` 的操作文件。

上述场景中导入的数据里 `S_ACTION` 菜单权限的 `PERMISSION_ID` 未正确设置，需要完成跨模块操作。

## Changes

### 根本原因

原有单阶段加载逻辑按模块顺序处理，每个模块内部先加载 PERM.yml 再加载 action yml。当 MODULE B（含 action yml）先于 MODULE A（含 PERM.yml）被扫描时，`identifierToPermId` 缓存为空，导致 `PERMISSION_ID = null`。

### 变更文件

**`BuildPermLoader.java`**

- `loadResources()` 改为两阶段调用：
  - 第一阶段：遍历所有模块，只扫描加载 PERM.yml → 填充 `identifierToPermId` 缓存
  - 第二阶段：遍历所有模块，加载 action yml → 此时所有 PERM.yml 已就绪，`permissionId` 可正确关联
- 新增 `loadPermissionsOnly()` + `scanPermissionsRecursive()` — 第一阶段入口
- 新增 `loadActionsAndResources()` + `scanActionsRecursive()` — 第二阶段入口
- 新增 `missingPermDirs` 集合：收集三级目录中缺少 PERM.yml 的路径（仅三级目录检查，一二级不检查）
- 新增 `getMissingPermDirs()` getter
- `findPermissionIdByIdentifier()` 去除逐条 warn 日志，改为写入 `missingPermDirs`
- 废弃 `loadResourceFromUri()` 和 `loadResourceRecursive()`（标记 `@Deprecated`）

**`BuildPerm.java`**

- 获取 `missingPermDirs`，在导入统计阶段统一打印缺少 PERM.yml 的目录汇总
