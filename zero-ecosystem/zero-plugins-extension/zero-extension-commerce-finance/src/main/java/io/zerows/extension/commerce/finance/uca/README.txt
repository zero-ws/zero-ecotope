此处加一个 README.txt 的文本对四个包的组件和职责做个详细说明
1. account
   主管账本，主要是账单和账单明细在执行操作过程中，提供一个自定义组件对账本中的记录执行变更。

2. enter
  （一般不访问数据库，数据库访问最多访问 X_NUMBER，其他层面不访问数据库）
   对象构造器，这个包中大部分的输入都是 JsonObject / JsonArray，通过对这两种请求对象执行处理，
   来完成对象本体的构造，如：账单、账单明细、转账明细、应收、应退、结算单、交易 等。
   还有一种特殊的情况是请求本身并非 JsonObject 结构，而是类似 String 的主键，可加载主键关联的
   数据，并且将对象从数据库中提取出来执行转换。

3. replica
  （一般不访问数据库，数据库访问只处理读，不执行写操作）
   对象复制器，对象复制器主要是针对两个对象：T / T 来执行：单对单、多对多、单对多、多对单的多重拷贝
   用于设置关联相关的关键属性，不做多余属性的设置，此处所有的基础属性都在 enter 中提前完成，包括序列化
   的动作也直接在 enter 包中的组件内完成。

4. trans
   交易模型，交易模型做的事主要是负责：单量、批量 兼容，它可以同时支持 JsonObject / JsonArray
   两种不同的结构，真正意义上完成：单对单、多对多、单对多、多对单的多重拷贝，并且只有下边两种组件可以
   访问数据库。
   - Service 组件：交易模型的上层组件
   - Trade 组件：交易模型内部组件
